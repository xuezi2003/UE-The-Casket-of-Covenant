# 关卡2 逻辑之匣 (L_Logic)

## 核心玩法

- **场景**：八边形中央区域 + 8个小房间
- **视角**：第三人称
- **核心**：答题 → 信息博弈 → 生死抉择

## 场景结构

```
      [1]
   [8]   [2]
 [7]  中央  [3]
   [6]   [4]
      [5]
```

- 中央区域：答题台
- 8个小房间：每轮4个安全 / 4个危险

## 游戏流程

### 单轮流程

```
1. 答题者上前答题（顺序轮流）
2. 题目答案为整数
3. 答对 → 告知4个安全房间 + 获得1位密码
   答错 → 无提示（差值用于后续房间伤害计算）
4. 答题者宣布结果（可欺骗）："我不知道" 或 "X号安全"
5. 限时进入小房间（不进 → 死亡）
6. 门锁前可自由进出（舔包）
7. 门锁上，危险房间放箭（伤害 = 个位差值 × 5）
8. 下一轮
```

## 惩罚机制

### 房间惩罚

- 进入危险房间 → 被箭射中
- 伤害 = 个位差值 × 5
- 示例：答案27，回答23 → 差值|7-3|=4 → 扣20HP
- 答对（差值=0）→ 无伤害 + 告知安全房间 + 获得1位密码

## 胜利条件

- **6位密码机制**
- 团队累计答对6题 → 获得完整密码 → 解锁出口
- 密码进度全队共享

## 开始/结束条件

**开始条件**：所有玩家加载完成，AI 最先生成

**结束条件**（满足任一）：
- 获得6位密码，存活玩家通关
- 全员死亡
- 真人全死（只剩AI）→ 结束整局
- 整体时间限制

## 策略要素

- 答题者可以撒谎，信任博弈
- 观察他人选择，推理安全房间
- 门锁前可舔包

## 金币系统

- 门锁前可进入房间舔包（只有金币）
- 答题者可用金币购买提示（不是答案）

## 待定系统

- 题目具体内容
- 提示具体内容
