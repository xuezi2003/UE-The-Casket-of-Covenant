## 十二、Content 文件夹结构

> **规则**：所有自己编写的资源统一放在 `Content/0_/` 下，与插件/引擎/示例内容隔离

### 设计原则

**Core + Level 两层隔离**：
- **Core/** — 核心框架 + 通用资源（跨关卡共用）
- **Level/** — 关卡专属内容（每个关卡完全独立）

```
Content/
└── 0_/
    │
    ├── Core/                      # ═══ 核心（跨关卡通用）═══
    │   ├── Framework/             # 框架基类（GI/GM/GS/PS/PC/AIC）
    │   ├── Character/             # 角色（Mesh/Anim/Material）
    │   ├── GAS/                   # 能力系统（Attribute/Ability/Effect/Cue）
    │   ├── Input/                 # 输入（IA/IMC）
    │   ├── UI/                    # 通用界面
    │   ├── Item/                  # 通用道具
    │   ├── Component/             # 通用组件
    │   ├── StateMachine/          # 通用状态机
    │   ├── VFX/                   # 特效
    │   ├── Audio/                 # 音频
    │   └── Data/                  # 数据资产（Enum/Struct/DataTable）
    │
    └── Level/                     # ═══ 关卡（各自隔离）═══
        │
        ├── MainMenu/
        │   └── UI/
        │
        ├── Lobby/
        │   ├── Framework/         # GM/PC 子类
        │   └── UI/
        │
        ├── Endurance/             # 耐力之匣
        │   ├── Framework/         # GM/GS 子类
        │   ├── Actor/             # 关卡专属 Actor
        │   ├── Item/              # 关卡专属道具
        │   ├── Component/         # 关卡专属组件
        │   ├── StateMachine/      # 关卡专属状态机
        │   └── GAS/               # 关卡专属能力/效果
        │
        ├── Logic/                 # 逻辑之匣（同上结构）
        ├── Courage/               # 勇气之匣
        ├── Insight/               # 洞察之匣
        ├── Sacrifice/             # 牺牲之匣
        │
        └── Dev/                   # 开发测试
```

### Core vs Levels 判断标准

| 问题 | 放 Core | 放 Levels/关卡名 |
|:-----|:--------|:-----------------|
| 多个关卡会用吗？ | ✅ | ❌ |
| 是基类吗？ | ✅ | ❌ |
| 是某关卡的子类/专属逻辑？ | ❌ | ✅ |
