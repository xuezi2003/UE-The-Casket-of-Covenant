# AI 助手行为指引

> 本文档定义 AI 助手在本项目中的行为规范与工作流程。

## ⚠️ 关键警告

### GameplayTag 操作规范
- **严禁自行添加或修改 GameplayTag**，必须经过用户二次确认
- 修改前**必须审阅**所有已使用该标签的位置（参考 `GameplayTags.csv` 及相关设计文档）
- 只在实际用到时才添加新标签

### API 使用规范
- 使用插件功能前**必须查阅官方文档**，严禁凭空捏造 API
- 首次查阅新插件后，将 URL 记录至 `参考文档/Plugin Documentation Links.md`

## 工作流程

### 文档先行
- 制定任何计划前，必须深度阅读相关设计说明书
- 收到「阅读文档」指令时，通览项目目录，精准识别所有关联章节

### 由博返约
1. 先确立全局实现架构与策略
2. 再详细拆解模块及其步骤
3. 最后分步执行

### 进度存证
- 及时更新 `待办文档/` 目录下的文件
- 确保开发进度与状态随时可追溯、可验证

## 设计原则

### 架构至上
- 严禁为追求进度而妥协整体架构设计
- 禁止为短期编码便利而牺牲系统可扩展性
- 所有系统设计必须预留扩展接口

### 拒绝临时代码
- 严禁为验证功能而编写零散的临时 Demo
- 每一行代码都必须是最终系统实现的一部分

### 工具效能最大化
- 面对繁琐问题优先思考技术最优解
- 充分挖掘现有插件、外部工具及搜索引擎的潜力
- 深度利用引擎与插件既有特性，框架及插件功能具有最高优先级

### 联网规范
- 始终遵循联网游戏开发范式
- 编写任何逻辑时必须考虑：数据同步、网络延迟、权限授权

## 文档规范

### 统一性
- 确保各文件间信息准确、同步
- 严禁出现逻辑矛盾或内容冗余

### 及时重构
- 当文档内容过长、职责过杂时，立即进行物理拆分
- 利用文件夹结构进行逻辑化管理

### 术语精确
- 严格区分函数与事件
- 描述业务流或逻辑跳转时，禁止使用含糊不清的词汇
