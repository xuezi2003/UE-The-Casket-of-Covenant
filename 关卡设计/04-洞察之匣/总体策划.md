# 关卡4 洞察之匣 (L_Insight)

## 核心玩法

- **场景**：无限循环走廊
- **视角**：第一人称
- **核心**：观察异常 → 站队判断 → 分组博弈

## 场景结构

```
Level
├── 等候区域 (0, 0, 0) - 通关玩家等待处
└── 走廊区域 (动态生成)
    └── 每个玩家独立走廊（位置 = 玩家编号 × 10000）
```

## 游戏流程

### 进入关卡

1. 为每个玩家动态生成独立走廊
2. 首轮所有玩家为同一组（看到相同异常）

### 单轮流程

```
进入走廊 → 铁门关闭
    ↓
走一段距离 → 站队区域出现
    ↓
观察走廊是否有异常
    ↓
限时站队（正常区/不正常区）
    ↓ 超时未站队 → 死亡
公布结果
    ↓
正确方向铁门打开
    ↓ 限时进入，超时 → 死亡
进入 → 铁门关闭 → 销毁旧走廊 → 生成新走廊
```

### 分组机制

- 上轮选择相同的玩家 → 同一组
- 同组玩家本轮看到相同异常
- 投票板显示：总人数 / 选正常 / 选不正常

### 通关流程

- 连续8次正确 → 白光转场 → 瞬移到等候区域
- 所有存活玩家到达等候区 → 进入下一关

## 开始/结束条件

**结束条件**（满足任一）：
- 所有存活玩家通关（到达等候区）
- 通用结束条件见 [通用逻辑](../00-通用逻辑/总体策划.md#关卡结束条件)

## 核心数据

| 数据 | 说明 |
|:-----|:-----|
| 连胜计数 | 连续8次正确即通关 |

## 惩罚机制

- 站队超时 → 死亡
- 不进入新走廊超时 → 死亡
- 答错 → 扣HP（具体数值待定）
- HP归零 → 死亡

## 技术要点

- 走廊动态生成，物理隔离（位置 = 玩家编号 × 10000）
- 异常配置由服务器决定，RPC 同步给客户端
