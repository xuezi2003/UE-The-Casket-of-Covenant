# 关卡1 测试环境 (L_Dev_Endurance)

## 概述

用于快速测试关卡1机制的开发环境，跳过状态树流程，通过触发体积手动控制游戏状态。

## 资产位置

```
Level/1_Dev_Endurance/
├── L_Dev_Endurance          # 测试关卡
└── BP/
    └── Framework/
        ├── GM_Dev_Endurance # 测试 GameMode
        └── GS_Dev_Endurance # 测试 GameState
```

## 核心类

### GM_Dev_Endurance

| 配置项 | 值 |
|--------|-----|
| 父类 | GM_Endurance |
| 游戏状态类 | GS_Dev_Endurance |

**BeginPlay**：不调用 Super，跳过状态树启动

**GetSpawnPos**：返回固定坐标 (0, 0, 0)

### GS_Dev_Endurance

| 配置项 | 值 |
|--------|-----|
| 父类 | GS_Endurance |

**BeginPlay**：不调用 Super，跳过状态树启动

**保留变量**：
- IsRedLight（红绿灯状态）
- IsDetecting（检测状态）
- 所有事件分发器

## 触发体积

| 体积 | 功能 | 状态 |
|------|------|------|
| BP_Vol_SetRedLight | 进入红灯/退出绿灯 | ✅ 已实现 |
| BP_Vol_TriggerStagger | 触发失衡（QTE 测试） | ✅ 已实现 |

**备注**：其他触发体积按需添加，AI 测试使用专门的测试 NPC + 简化行为树。

## 使用方法

1. 打开 `L_Dev_Endurance` 关卡
2. 运行游戏
3. 走进不同触发体积测试对应功能

## 与正式关卡的区别

| 对比项 | L_Endurance | L_Dev_Endurance |
|--------|-------------|-----------------|
| 状态树 | 自动运行 | 不运行 |
| 红绿灯 | 状态树控制 | 触发体积手动控制 |
| AI 填充 | FillAI 阶段 | 触发体积手动生成 |
| 出生点 | BP_BornVol 随机 | 固定坐标 |
