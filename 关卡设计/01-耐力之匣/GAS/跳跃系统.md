# 跳跃系统

**父类**：GSCGameplayAbility

## 变量

| 变量名 | 类型 | 值 | 说明 |
|--------|------|-----|------|
| StaminaCost | Float | 15.0 | 用于激活前体力检查 |
| Effect Container | Array | - | 保存应用的 GE Handle |

## 标签配置

| 配置项 | 值 |
|--------|-----|
| 资产标签 | `Player.Action.Jumping` |
| 激活阻止标签 | `Player.Action.Jumping`, `Player.State.Exhausted`, `Player.State.Staggered`, `Player.State.Fallen`, `Player.State.Dead`, `Player.State.Attacked` |

**设计说明**：
- `Player.Action.Jumping` 在激活阻止标签中 → 防止二段跳
- `Player.State.Exhausted` → 力竭时无法跳跃
- 不需要 `Ability.Require.Stamina` → 一次性消耗技能不会被力竭强制取消

## 事件图表

### Event ActivateAbility

```
CheckStamina (函数) → 返回 Bool (Stamina >= Abs(StaminaCost))
    ├─ False → DelayUntilNextTick → End Ability（不扣体力，延迟一帧避免 BTTask 卡死）
    └─ True ↓
CommitAbility
    ↓
Apply Effect Container (Effect.Container.Jump) → 保存到 Effect Container 变量
    ↓
Get Avatar Actor → Cast To Character
    ↓
Character.Jump()
    ↓
Wait Input Release (IA_Core_Jump)
    ├─ then: Bind CMC.OnMovementModeChanged → HandleMovementChanged
    └─ 发布时: Character.StopJumping()
```

### CheckStamina 函数

**设计说明**：
- **一致性**：为代码一致性和未来扩展性，跳跃系统也实现了激活前体力检查。
- **详细实现与原理**：详见 [体力系统.md](体力系统.md#激活前体力检查checkstamina-函数)。

### HandleMovementChanged（自定义事件）

参数：Character, PreviousCustomMode, PrevMovementMode

```
PrevMovementMode = Falling?
    └─ True → End Ability
```

**说明**：当前一个状态是 Falling，说明刚落地，结束技能。

### Event OnEndAbility

```
Get Avatar Actor → Cast To Character → StopJumping()
    ↓
For Each (Effect Container Handles)
    → RemoveGameplayEffectFromOwnerWithHandle
    ↓
Array Clear (Effect Container)
```
